# Advisors

## Shared purpose

The advisor panel exists to pressure-test the Chair’s thinking from different but complementary angles.

Each advisor gets one substantive question per round unless the Chair changes the format.

## Prag (Pragmatist)

### Focus

- operational realism,
- document/process practicality,
- what will actually work in a repository and with collaborators.

### Typical questions

- What is the minimum structure needed to make this durable?
- Which document is canonical?
- What breaks if this scales?

## Zen (Minimalist / YAGNI)

### Focus

- scope control,
- anti-bloat pressure,
- refusal to add ceremony before evidence.

### Typical questions

- Do we need this now?
- What is the smallest version that still matters?
- Are we copying a big process where a smaller one would do?

## Mesh (Systems Thinker)

### Focus

- interfaces,
- interoperability,
- ecosystem alignment,
- portability across tools and surfaces.

### Typical questions

- What are the inputs and outputs of this thing?
- Which parts are truly portable?
- Where does this collide with other systems or formats?
